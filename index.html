Description

Data Preprocessing：

1. 以 Set cover Optimization 演算法講資料集中的點縮減至 10000 筆

2. 將玩家根據其名次進行分級（Ex. 1-10 名為等級 1）

3. 由於資料為死亡玩家資料，因此藉由抓出名次為 2 之玩家，其 Killer Position 即吃雞玩家獲勝地點

Overview：

1. 在地圖上以紅點標示玩家死亡位置，鼠標移動至點上可查看該玩家之詳細資訊

2. 拉動地圖下方時間軸，可動態呈現遊戲時間及死亡分佈狀況的關係

3. 利用右方選項欄位，根據不同地圖、武器或死亡玩家等級進行篩選調整

4. 由右方的名次長條圖，可看出殺手及死亡玩家的等級分佈，且亦會隨篩選條件變動

5. 在下方區塊的地圖上，可看出最終獲勝玩家的位置分佈

6. 亦可利用將鼠標移動至點上查看獲勝玩家姓名及最終使用的武器

Insight：

1. 衝突區（死亡點密度高的地方）幾乎都在大城，但吃雞點通常不是在大城且吃雞地點分布平均

2. 拖拉時間軸可發現較早的死亡的那批玩家通常死在大城，可以看出玩家一開始跳傘較愛跳大城，也是物資比較多的地方

3. 玩家等級較前面的死亡資料，武器 M416 佔最多數

4. 時間軸前期和等級較低的玩家則不分武器，因為通常一落地撿到槍會直接開打，較不會管自己撿到什麼武器

5. 從 Killer Placement 可看出名次高的人殺的也較多，代表殺越多人吃雞的機率越高，所以我們認為一開始跳大城拿物資拼槍是值得冒險的（也可以順便練槍法）
